A new paper called Meta-Harness: End-to-End Optimization of Model Harnesses1 builds on a thesis that is getting harder to ignore: agent performance is no longer just about the model. It is also about the harness — the layer that decides what the model sees, what gets stored, what gets retrieved later, and how the loop is orchestrated. The paper’s claim is that harness optimization may be as important as model optimization. I think it’s right.

Codex and Claude Code are the clearest examples of this in practice. Both wrap the model with sophisticated harnesses tuned across instructions, tools, memory and context management, and safety controls — they make the system far more useful than the raw model alone. Labs are post-training their models against their own harnesses, which is why a model like GPT‑5.4 loses ground when you take it outside Codex and into envs like Cursor or Kilo Code.

What makes Meta-Harness interesting is that it stops treating the harness as fixed. The architecture is easiest to read as two nested loops: an inner harness that runs the task and manages context, memory, and state; and an outer loop that acts as an optimization layer around it. The outer loop hands a coding agent prior harness candidates — source code, scores, and execution traces — so it can inspect failures, compare versions, and rewrite the harness itself.

Search over the harness, not over the model

What this really is, mechanically, is search over harness code with an observability layer over prior attempts. Harness failures are usually long-horizon context problems: a bad retrieval, storage, or presentation decision early on may only surface much later in the rollout. Rather than relying on scalar rewards or compressed summaries, Meta-Harness lets the agent selectively inspect raw history the way a strong engineer would — check the logs, inspect the traces, compare the code, isolate the failure, then change the scaffolding. In the paper’s setup, a single evaluation can generate up to ten million tokens of diagnostic information, which is exactly why selective access to raw history matters.

A primitive worth building

Meta-Harness is a strong start. It validates that there is real opportunity in building a harness system that evolves — one that observes its own failures, reasons over traces, and continuously rewrites its retrieval, memory, and orchestration policies without drifting away from the strengths of the underlying model. The best builders are already moving here. Whoever lands this primitive cleanly owns a meaningful piece of the next software paradigm.
